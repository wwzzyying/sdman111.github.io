自定义Cell不生效？

尝试在storyboard设置Collection View Flow Layout的Size inspector中的Estimate Size属性为none

数据源分类

主要是将数据改为嵌套数组的形式，然后根据子数组数目，即数组count来决定段数；然后在进行渲染是会根据我们返回的自定义数据在渲染过程中进行拦截，在回调函数中根据已知参数（已渲染的信息）进行配置。

配置补充视图Header

每个UICollectionView的Section Header或Footer是一个UICollectionReusableView，所以对应的datasource方法必须返回UICollectionReusableView类型，并且要给每个header或footer创建并连接上一个class类，将header或footer中的组件连接到对应类中，便于后期动态配置组件属性。

与cell的回调函数类似，渲染header或footer这类补充视图也会有对应的回调函数：

override func collectionView(_ collectionView: UICollectionView, viewForSupplementaryElementOfKind kind: String, at indexPath: IndexPath) -> UICollectionReusableView

具有补充视图或reusableview相关字眼。对于补充视图的渲染类似于cell的方式，是通过identifier进行渲染的，相同identifier的渲染效率更高？所以我们对header设置identifier，然后根据回调函数中的kind参数进行判断：

因为kind是String类型，所以UICollectionView中一定有对应kind的String

如果是header，即UICollectionView.elementKindSectionHeader

如果是footer，即UICollectionView.elementKindSectionFooter

CollectionView布局的相关属性

cell默认方向为水平方向，当该行放不下下一个cell时自动换行

1、Min Spacing：每一行内各个item或每一行之间的最小距离

If there is extra space at the end of the line, the layout object increases the interitem spacing until the items fit evenly within the line boundaries

如果有额外的空间将会被均匀分配

2、Section Insets：段的内边距，类似于HTML中的Padding，通过let inset = UIEdgeInsetsMake(top, left, bottom, right)进行构建

动态调整cell

CollectionViewController直接遵循UICollectionViewDelegateFlowLayout协议，因为已经自动设置delegate为self，所以仅需要在vc遵循该协议即可。

UICollectionViewCell视图层级

从cell到组件的层级依次是cell -> backgroundView -> selectedBackgroundView -> contentView

backgroundview和selectedBackgroundView默认不显示，需要手动添加；

而默认仅有contentView

依次显示Cell -> backgroundView -> selectedBackgroundView -> contentView -> label

是UICollectionViewDelegate中的方法

动态创建CollectionView

流程就是实例化UICollectionView并设置dataSource委托、delegate委托，view调用addsubview添加CollectionView，利用下面的init方法，

init(frame: CGRect, collectionViewLayout layout: UICollectionViewLayout)，创建的重点在于layout参数不能简单实例化一个UICollectionViewLayout并赋值就完事了，因为UICollectionViewLayout是一个抽象类，我们需要实例化UICollectionViewLayout的子类UICollectionViewFlowLayout，然后初始化flowLayout的属性，包括最小行距、最小间距、内边距等，然后传给CollectionView的init方法即可。

Registers a class for use in creating new collection view cells.

然后CollectionView一定要使用register方法注册cell，并创建继承UICollectionViewCell的子类
