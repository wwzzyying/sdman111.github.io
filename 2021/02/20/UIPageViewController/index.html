自定义PageControl

解决系统默认的大黑边问题

原因是因为pagecontrol分页控件默认实现是添加在pagevc分页视图控制器上面的，而分页控件背景透明，分页视图控制器默认黑色。那为什么显示的vc不会覆盖掉这个分页控件呢？通过观察层级关系，分页控件在pagevc中默认占据底部一部分位置，后面setvc所添加的vc与这个分页控件是共用这个pagevc的。

解决思路

不重写两个presentation函数，通过自定义分页控件并添加在视图最上方来避免显示pagevc默认的黑色背景问题。

开始自定义分页控件

大概步骤

声明pageControl全局变量 -> 实例化UIPageControl类并赋值给变量 -> 初始化numberOfPages属性，因为currentPage属性默认为0所以不设置 -> 然后最重要的一步是给view添加subview，使得分页控件在view最上方

前提

遵循UIPageViewControllerDelegate协议并设置delegate委托

查看文档，官方说明：

These methods allow the delegate to receive a notification when the device orientation changes and when the user navigates to a new page

表明delegate下面的方法是在设备方向发生变化或者用户滑动到新的页面之后对这些变化发出的通知进行接收的方法，所以我们需要拦截这些方法的默认实现，使得vc随着滑动更改时会修改分页控件的currentPage属性。
