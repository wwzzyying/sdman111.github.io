半年后的杂谈

2021-10-16

距离上次发文章已经快半年了，这半年一次写技术博客的冲动都没有，倒也不是没东西可写，只是单纯懒了哈哈哈哈，难得周末有空就写一下杂谈吧！

刚开始实习时

因为一直都是写 OC 和 Swift 嘛，我以为大公司维护一个 iOS 项目也是只需要写业务就行，没想到进去的第一天就得知小组原来是分为基础组和业务组，我归到基础组来进行实习，另一位同学归到业务组。直观上区别就是我平时的工作内容就是使用 ruby 来写脚本，业务的同学使用 OC/Swift 来写 Demo；之所以使用 ruby 是因为 cocoapods 这个包管理工具就是用 ruby 写的，并且提供了许多内置的类似 xcodeproj 的内置库来修改项目，可以直接修改项目构建设置等等。

前一两个月因为实习生不能看项目源码，所以只能做一下围绕提高项目开发效率或者分析构建产物的工具，旨在优化项目、定位可能出现的问题；

第一个实践

第一个接触的实践就是使用 ruby 写一个分析构建产物 LinkMap 的脚本，主要目的是生成每个类与链接进最终产物的该类的函数的索引，然后还实现包体积分析、包含的 bundle 包体积、每个通过 pod 引入的组件链接进最终产物的体积、每个 .o 目标文件链接进最终产物的符号体积、通过类和函数的索引来搜索某个函数所在的目标文件和所在类、寻找被分类重写的方法并且输出所在位置、区分 development pod 和 remote pod（需要 hook 一下 post_install）。

URLSession网络请求

get请求

第三方库AFNetworking的桥接与使用

2021-03-10

Swift桥接使用OC第三方库

  1. 在项目中新建Group用于放置第三方库需要桥接的文件，勾选create groups
  2. 根据系统提示创建桥接文件；如果在之后的操作中误删桥接文件只需要在原位置创建文件名格式为：Swift Target名-Bridging-Header.h的桥接文件即可；如果桥接文件位置发生变化则到targets中找到项目-Build Settings-Swift Complier General中修改Objective-C bridging header文件相对于项目的相对路径即可
  3. 在桥接文件中引入第三方库的头文件：#import “AFNetworking.h”即可在Swift项目中使用第三方OC库

2021-02-03

框架

利用CoreBluetooth框架实现应用程序与低功耗蓝牙（Bluetooth low energy，简称BLE）设备之间的通信，

1、The framework is an abstraction of the Bluetooth 4.0 specification for use with low energy devices.

该框架是蓝牙4.0规范的抽象，适用于低功耗设备。

换言之，蓝牙规范低于4.0的苹果设备就不适用该框架

2、An iOS app linked on or after iOS 10.0 must include in its Info.plist file the usage description keys for the types of data it needs to access or it will crash. To access Bluetooth peripheral data specifically, it must include NSBluetoothPeripheralUsageDescription.

使用该框架的软件必须在info.plist中添加对蓝牙外部设备的使用说明用来告知用户是否赋予该权限，否则应用将崩溃。
