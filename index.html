URLSession网络请求

2021-03-10

get请求

最常见的一种请求方式，当客户端要从服务器中读取文档时，当点击网页上的链接或者通过在浏览器的地址栏输入网址来浏览网页的，使用的都是GET方式。GET方法要求服务器将URL定位的资源放在响应报文的数据部分，回送给客户端。使用GET方法时，请求参数和对应的值附加在URL后面，利用一个问号（“?”）代表URL的结尾与请求参数的开始，传递参数长度受限制。

缺点

  • 不方便传输大量数据
  • 不适合传输私密数据

第三方库AFNetworking的桥接与使用

2021-03-10

Swift桥接使用OC第三方库

  1. 在项目中新建Group用于放置第三方库需要桥接的文件，勾选create groups
  2. 根据系统提示创建桥接文件；如果在之后的操作中误删桥接文件只需要在原位置创建文件名格式为：Swift Target名-Bridging-Header.h的桥接文件即可；如果桥接文件位置发生变化则到targets中找到项目-Build Settings-Swift Complier General中修改Objective-C bridging header文件相对于项目的相对路径即可
  3. 在桥接文件中引入第三方库的头文件：#import “AFNetworking.h”即可在Swift项目中使用第三方OC库

iOS蓝牙开发基础

2021-02-03

框架

利用CoreBluetooth框架实现应用程序与低功耗蓝牙（Bluetooth low energy，简称BLE）设备之间的通信，

1、The framework is an abstraction of the Bluetooth 4.0 specification for use with low energy devices.

该框架是蓝牙4.0规范的抽象，适用于低功耗设备。

换言之，蓝牙规范低于4.0的苹果设备就不适用该框架

2、An iOS app linked on or after iOS 10.0 must include in its Info.plist file the usage description keys for the types of data it needs to access or it will crash. To access Bluetooth peripheral data specifically, it must include NSBluetoothPeripheralUsageDescription.

使用该框架的软件必须在info.plist中添加对蓝牙外部设备的使用说明用来告知用户是否赋予该权限，否则应用将崩溃。

自定义iOS转场动画

2021-01-28

构造流程

To present a view controller using custom animations, do the following in an action method of your existing view controllers:

  1. Create the view controller that you want to present.
  2. Create your custom transitioning delegate object and assign it to the view controller’s transitioningDelegate property. The methods of your transitioning delegate should create and return your custom animator objects when asked.
  3. Call the presentViewController:animated:completion: method to present the view controller.

When you call the presentViewController:animated:completion: method, UIKit initiates the presentation process. Presentations start during the next run loop iteration and continue until your custom animator calls the completeTransition: method. Interactive transitions allow you to process touch events while the transition is ongoing, but noninteractive transitions run for the duration specified by the animator object.

第一步：获取已经创建好的所需要present的VC。

第二步：创建一个遵循UIViewControllerTransitioningDelegate协议的类，并把该类对象赋值给需要present的VC（即第一步中的VC）的transitioningDelegate属性；遵循UIViewControllerTransitioningDelegate协议的这个类所需要实现的方法必须返回所创建的自定义动画对象animator object。
