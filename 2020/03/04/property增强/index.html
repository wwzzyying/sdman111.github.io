@property增强

2020-03-04

包括@property、@synthesize和@property增强

@property

这里是指Xcode4.4版本之前的@property

作用

自动生成getter、setter方法的声明，写在@interface之中

语法

原理

编译器在编译时，会根据@property生成getter和setter方法的声明；

注意

  1. @property的类型和属性的类型一致；

    @property的名称和属性的名称一致(去掉下划线)；

  2. @property的名称决定了生成的getter和setter方法的名称；

    @property的数据类型决定了生成的getter方法的返回值类型、setter方法的参数类型。

批量声明

@property float height,weight;

​ 前提是数据类型相同

@synthesize

作用

自动生成getter、setter方法的实现，写在类的实现中；

语法

synthesize做的事情

  1. 用@synthesize @property的名称会自动生成一个私有属性(不带下划线);

  2. 自动生成setter方法的实现；

    实现方法：将传入参数赋值给那个私有属性，没有任何逻辑验证；

  3. 自动生成getter方法的实现；

    实现方法：直接返回该私有属性；

此时，声明中的属性_age失去意义，可以使用另一个语法替换它

语法

@synthesize @property名称 = 已经存在的属性名;

@synthesize age = _age;将属性的setter,getter方法，作用于这个变量

此时：

  1. 不会再生成私有属性；
  2. 直接生成实现，但操作的对象变成指定的属性；

批量实现

@synthesize name,age,weight;

或

``@synthesize name = _name,age = _age,weight = _weight;`

不需要数据类型相同；

@property增强

在Xcode4.4之后，编译器引入property autosynthesis，即属性自动合成；

@property int age编译器会自动：

  1. 生成私有属性(自动加上下划线)；
  2. 生成getter、setter方法的声明；
  3. 生成getter、setter方法的实现。

注意

  1. @property生成的方法同样没有任何逻辑验证；

  2. 可以通过重写getter或者setter来满足需求，但不能同时重写两个方法，否则编译器不会自动生成私有属性；

  3. 此时编译器会报错，因为没有这个属性，此时我们可以自己在声明中自己写属性解决。

继承

父类的@property一样能被子类继承，但由于属性为父类私有，不能直接访问，只能通过getter、setter方法访问；

[super setAge:100];
