Objective-C笔记(6)

2020-02-17

包括nil和NULL的异同，同类型指针相互赋值，分组导航标记，函数和方法的异同。

nil和NULL

  • 都只能作为指针变量的值；
  • 都表示不指向任何一块内存空间；
  • 实际上都是宏，都等价于0；
  • 本质上一样。

但是在使用上还是要有区分：

  • C指针用NULL

int *p1 = NULL，表示p1指针不指向内存中任何一块空间；

  • OC类指针用nil

Person *p1 = nil，表示p1指针不指向任何对象；若此时再访问p1指针指向对象的属性则会报错，而访问方法则不会报错，但不会执行。

同类型指针变量之间可以相互赋值

1、

Person *p1 = [Person new];

Person *p2 = p1;

p1和p2的类型都是Person指针类型，p1和p2指向堆中的同一个地址，表示指向同一个对象。

2、

Person *p2 = p1;

p2 = [Person new];

此时p2会重新指向一块在堆中新建的对象。

分组导航标记

1、#pragma mark 分组名

2、#pragma mark -

​ 产生水平横条

3、#pragma mark - 分组名

​ 产生水平横条后显示分组名

函数和方法

  • C语言中的函数
  • OC中的方法

相同点：封装一段代码，表示一个相对独立的功能；当函数或方法被调用时，封装在其中的代码会被自动执行。

不同点：

  • 语法不同；
  • 定义的位置不同；

//OC方法声明只能写在@interface的大括弧之外，方法实现只能写在@implementation中；

//函数除了在函数内部和@interface大括弧之中，其他地方都可以写；

//就算把函数写在类中，函数仍不属于类，所以类创建的对象中没有这个类；

//把函数写到类中是不规范的，不要这样写。

  • 调用方式不同

函数直接调用；test();

方法要通过类创建对象再通过对象调用。
