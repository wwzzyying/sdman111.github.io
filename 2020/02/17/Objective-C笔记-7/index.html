Objective-C笔记(7)

2020-02-17

包括类声明实现常见错误，多文件开发，等等。

常见错误

  • 类要先声明后实现；
  • 声明和实现都要有，就算声明没有方法也要写出实现；
  • 声明的位置必须在使用类前，实现可以放在使用类后；
  • 属性不允许声明时初始化；
  • 当方法只有声明没有实现会报Method definition for 'xxx' not found警告，而当通过对象调用方法并运行时会报错，而不是像类指针为nil时调用方法不报错。

多文件开发

为什么要多文件？

——为了避免所有类的声明和实现写在同一个实现文件导致后期维护困难。

所以，最好把类的声明和实现分开写，一个头文件写声明，一个实现文件写实现；头文件和实现文件写在同一个Target下。

  • .h头文件由于要写声明要引用NSObject类，所以要引入<Foundation/Foundation.h>
  • .m实现文件要先#import “xxx.h”引入声明部分。

这是单独创建两个文件，而更快的是在Target下NewFile-Cocoa Class，直接自动生成.m和.h文件，并把声明和实现部分的语法自动填充好。

  • 创建Cocoa Class文件的名称是模块文件，也就是两个文件的名称；
  • 模块文件里面的类名可以修改，但为了方便管理，不建议修改。

对象和方法

  1. 对象作为方法的参数：也就是参数类型是类指针，传递的是地址：- (void)test:(Dog *)dog;；所以方法内部通过形参去修改形参指向的对象时，会影响实参变量指向的对象的值。
  2. 类的本质是自定义的数据类型，而数据类型是在内存中开辟空间的模板。
  3. 对象作为方法的返回值：相应地用另一个类指针保存指针值。另外，若返回值类型是当前类对象，最好用instancetype代替。

一个Target中的类无法直接在另一个Target中访问

解决：直接把需要的类所在的文件copy过去，show in Finder—拖动文件到相应Target—勾选copy items if needed和create groups即可。

对象作为类的属性

所谓属性，就是这个类拥有的特征。

一个人类，一个狗狗类，人也可以拥有一个狗狗对象，此时就把狗狗作为人的一个属性；

而这个_dog属性在堆中仅仅是一个指针变量，初始值为nil，而不是一个对象；

此时p1->_dog->_name="..."或者[p1->_dog shout]无论访问属性或者调用方法都会报错，因为此时_dog值为nil；

解决：可以创建另一个狗狗对象，然后把这个对象的值(地址)赋值给_dog变量。

  • A类可以作为B类的属性，代表B拥有A；
  • 属性仅仅是一个变量而已，一个指针变量而已，默认值是nil，并没有创建对象。
