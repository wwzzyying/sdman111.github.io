OC异常处理

2020-02-21

异常

程序可以编译、链接、运行，当程序运行时，出于某些条件下，程序执行终止。

异常后果

程序立即崩溃，立即终止运行，且后面代码不执行。

异常处理

目的

为了让程序发生异常时不崩溃，继续往下执行。

语法

当发生异常时，执行@catch块，然后继续往下执行。

当不发生异常时，执行@try块，然后略过@catch块，继续往下执行。

  • @catch中一般写处理异常的代码；

  • 打印ex指向的对象的值，拿到异常原因。
@catch(NSException *ex)
{
    NSLog(@"%@",ex);
}

C语言的异常无法处理
int num1 = 10;
int num2 = 0;
int num3 = num1/num2;
这时候try不起作用，运行时还是会发生异常然后崩溃。

@try...@catch不是万能的
例如：
@try
{
    Person *p1;
    p1->_name = @"xxx"；
}
此时无法捕捉异常

所以，处理异常更多是逻辑上的判断
Person *p1;
if(p1 != nil)
    p1->_name = @"xxx";
