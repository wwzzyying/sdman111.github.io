Java正则表达式学习随笔

2019-08-02

正则表达式将规定格式的表达式创建为一个模式，通过java.util.regex包中的类的静态方法和对象方法进行字符串的模式匹配，可用于数据验证或者提取子字符串等等。

查看API可知，java.util.regex包含有Pattern和Matcher类、PatternSyntaxException正则表达式语法错误异常类、和一个被Matcher实现类实现的接口MatchResult接口。

首先看一段典型的调用序列：

首先看Pattern(模式)：

因为Pattern类没有公共构造方法，所以只能通过Pattern类的公共静态编译方法compile(String regex)，将指定为字符串的正则表达式编译为模式，返回一个对象，这个对象是Pattern类的一个实例，并且被Pattern对象p引用。

Matcher(匹配器)：

创建Matcher匹配器对象m需要通过Pattern对象调用对象方法matcher(CharSequence input)创建一个匹配器并且返回这个实例给匹配器对象m。

PatternSyntaxException(模式错误异常类)：

非强制异常类，表示正则表达式模式中的语法错误。

要注意的是Java中的正则表达式与其他语言不同，Java表示\反斜杠需要\\两条反斜杠，这样才会具有转义意思；例如表示一位数字的正则表达式是\\d而不是\d;表示一个普通的反斜杠\需要写成\\\\

Matcher类的start()、end()和reset()方法

第一次匹配器调用matches()方法后检测字符串是否匹配，当索引检测到第3位即-，而matches()方法不同于find()方法的地方是要整个字符串区域与模式匹配才返回true，所以此处返回false，标记(光标)停留在第3位；接着执行find()方法（只要检测到一位符合即返回true），标记移动到下一位即第4位，此处为start()方法返回的起始索引，当移动到下一个-时，标记停留在第9位，此时调用end()方法返回结束时所处的偏移量。

匹配器调用reset()方法，标记置于字符串区域开始处，之后显示索引正常。

捕获组

捕获组是把多个字符当一个单独单元进行处理的方法，它通过对括号内的字符分组来创建。

例如，正则表达式 (dog) 创建了单一分组，组里包含”d”，”o”，和”g”。

捕获组是通过从左至右计算其开括号来编号。例如，在表达式((A)(B(C)))，有四个这样的组：

  • ((A)(B(C)))
  • (A)
  • (B(C))
  • (C)

正则表达式中以’()’标记的子表达式所匹配的内容就是一个分组(group)，分组索引是从1开始的,0代表正则表达式匹配的整个字符串,group(i)代表第i组匹配的内容。所以匹配器调用group(0)返回的是匹配的整个字符串区域，group(i)则是返回匹配的整个字符串区域的子表达式()括号内的内容

一开始我纳闷，为什么匹配出来的结果是John writes而不是John w

直到再一次认真看下面这个两个正则表达式的区别

原来是空格惹的祸，第一个运行时模式匹配John 加上一个由一个或多个字符（尽可能少）组成的组，但要直到遇到空格才结束，匹配到子字符串，匹配结束。

第二个运行时则是遇到John空格后面第一个字符即匹配结束，因为(.+?)为非贪婪模式，只要发现一个匹配就立马结束，即”John w”。

所以匹配器要严格按照模式进行匹配。
