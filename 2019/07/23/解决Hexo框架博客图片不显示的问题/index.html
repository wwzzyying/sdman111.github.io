解决Hexo框架博客图片不显示的问题

2019-07-23

第一种

直接使用markdown语法![Alt Text](url)，url为图片在电脑硬盘的绝对地址，但hexo框架在显示markdown语法插入的图片会出现图片不会正常显示的现象，hexo s之后博客会显示下面那样：

第二种

打开博客所在文件夹blog，找到_config.yml文件，修改post_asset_folder: true

每次执行hexo new "文件名"命令时，文件所在文件夹会同时创建一个与同名文件的文件夹，可以把文件所需资源放在此文件夹内。

打开你的md文件，使用标签插件来插入同名文件夹内的图片（！！！这是hexo3已写入核心代码的标签插件）

当我使用标签插件插入图片成功后，奇怪的事发生了，在我的md文件内标签插件仅仅正常显示了两张图片，其他插入均失效了，然后使用markdown语法![图片描述](/md文件夹名字/example.jpg)也不能正常显示。

如果你可以正常插入图片那可以不用看下去了

第三种

在blog/source目录下新建一个images目录，把所需插入的图片放进此目录，还是使用markdown语法![图片描述](/images/example.jpg),感觉与第二种方法大同小异，依旧不能成功插入。下一个下一个

第四种

在博客坐在目录blog调用git bash

调用命令npm install https://github.com/CodeFalling/hexo-asset-image --save等待片刻，完成安装后用hexo新建文章的时候会发现_posts目录下面会多出一个和文章名字一样的文件夹。图片就可以放在文件夹下面。

但是需要修改插件内容，不然会出Bug。打开/node_modules/hexo-asset-image/index.js，替换成下列内容：

'use strict';
var cheerio = require('cheerio');

// http://stackoverflow.com/questions/14480345/how-to-get-the-nth-occurrence-in-a-string
function getPosition(str, m, i) {
  return str.split(m, i).join(m).length;
}

var version = String(hexo.version).split('.');
hexo.extend.filter.register('after_post_render', function(data){
  var config = hexo.config;
  if(config.post_asset_folder){
      var link = data.permalink;
  if(version.length > 0 && Number(version[0]) == 3)
     var beginPos = getPosition(link, '/', 1) + 1;
  else
     var beginPos = getPosition(link, '/', 3) + 1;
  // In hexo 3.1.1, the permalink of "about" page is like ".../about/index.html".
  var endPos = link.lastIndexOf('/') + 1;
    link = link.substring(beginPos, endPos);
var toprocess = ['excerpt', 'more', 'content'];
for(var i = 0; i < toprocess.length; i++){
  var key = toprocess[i];

  var $ = cheerio.load(data[key], {
    ignoreWhitespace: false,
    xmlMode: false,
    lowerCaseTags: false,
    decodeEntities: false
  });

  $('img').each(function(){
if ($(this).attr('src')){
  // For windows style path, we replace '\' to '/'.
  var src = $(this).attr('src').replace('\\', '/');
  if(!/http[s]*.*|\/\/.*/.test(src) &&
     !/^\s*\//.test(src)) {
    // For "about" page, the first part of "src" can't be removed.
    // In addition, to support multi-level local directory.
    var linkArray = link.split('/').filter(function(elem){
    return elem != '';
    });
    var srcArray = src.split('/').filter(function(elem){
    return elem != '' && elem != '.';
    });
    if(srcArray.length > 1)
    srcArray.shift();
    src = srcArray.join('/');
    $(this).attr('src', config.root + link + src);
    console.info&&console.info("update link as:-->"+config.root + link + src);
  }
}else{
  console.info&&console.info("no src attr, skipped...");
  console.info&&console.info($(this));
    }
  });
  data[key] = $.html();
    }
  }
});

不出所料，又失败了。

第五种

使用图云，浏览器内安装“微博图床”扩展

选取电脑内的图片或者拖拽图片，生成Markdown连接，复制粘贴到md文件中，成功了！！！

前面几个方法都不成功，只有图床能完整并且全部显示插入的图片。

如果标签插件和安装插件都没有成功的话可以试一下图床扩展！
